Driver Monitoring System

In-cabin sensing · production-shaped prototype

Real-time psychophysiological inference with dual hazard topology: fatigue vs. visual distraction.

Public deployment: https://dms.aher.dev/

Architecture

The stack is deliberately split across the edge-adjacent client (all vision inference local to the browser—no raw frame egress) and a stateful control plane implemented as an async Python service. Telemetry egress is limited to derived scalar features, pose estimates, and quantized gaze proxies serialized in a versioned WebSocket schema (metrics_batch v1).

Ingestion uses FastAPI with ASGI lifecycle hooks for idempotent schema materialization (SQLAlchemy 2.0 declarative + create_all Phase-1 bootstrap; migration discipline deferred to Alembic). Persistence targets PostgreSQL via asyncpg in production with negotiated TLS to the wire protocol; local / CI fall back to SQLite + aiosqlite for zero-config ergonomics.

Perception substrate

  • MediaPipe Face Mesh (refined landmark topology, iris pentads enabled): per-frame extraction of six-point EAR / MAR analogs, rolling-statistics–friendly scalarization.
  • Head pose: iterative PnP against a canonical facial rig when OpenCV.js loads; deterministic geometric fallback to preserve availability under CDN flake or CSP friction.
  • Attention & distraction proxy: iris–sclera offset normalized against palpebral fissure extrema, fused with Euler-angle priors to emit a discrete gaze_region lattice (forward / lateral / off-boresight / ventral fixation).

Inference & policy

  • RollingSignalBuffer: server-monotonic deques with configurable horizon (~30 s @ 200 ms client batching cadence) feeding short-window feature extraction (EAR dominance fraction, MAR excursion counting, pitch swing / nodding heuristics).
  • Dual alert topologies: (a) debounced LLM synthesis via OpenAI Chat Completions with SHA-256–keyed snapshot deduplication and 30 s recall floor; (b) parallel DistractionMonitor with independent hysteresis, speed-gated activation, and region-majority voting over sliding server windows—orthogonal cooldown namespaces vs. fatigue channel.
  • Threshold fallback when the generative path is cold: deterministic severity lattice aligned with benchmark baselines (fixed EAR floor vs. contextual engine A/B in offline notebook pipeline).

Data plane

First-class trip entities mirror WebSocket session UUIDs: bounded route_json polylines, odometer integration from synthesized speed–time quadrature, monotonic alert counters, and append-only alert ledger rows for post-hoc forensics and dashboard backfills. REST projections expose cursorable trip lists and per-trip alert timelines.

Deployment

Containerized Python 3.11 image, Railway-native DATABASE_URL injection, PORT-bound Uvicorn workers, static asset hosting co-located with the API for single-origin WebSocket + HTTPS camera policy compliance.

Disclaimer
